## ❌ If Something Goes Wrong

### Dashboard won't load
```bash
# Check backend is running
curl http://localhost:5000/health

# Restart frontend
cd frontend && npm run dev
```

### Buttons don't work
- Check browser console (F12) for errors
- Check backend terminal for error logs
- Try refreshing the page

### Chat doesn't respond
- Wait 10 seconds (Groq API might be slow first time)
- Check backend logs for API errors
- Make sure Groq API key is set in `backend/api.py`

### Data looks stale
- Click "Refresh Analysis" button
- Or restart backend: `python backend/api.py`

## 💬 Q&A Handling

**Q: Is this real AI or mocked responses?**
> Real Groq Llama 3.3 70B. Every answer is generated by the LLM based on live data from SQLite. Watch the "Show reasoning" tab to see the exact steps taken.

**Q: How does it handle the multi-step reasoning?**
> The planner generates a sequence of steps (e.g., get demand history → forecast → check stockout risk). The orchestrator executes them in order, substituting results from earlier steps into later parameters. It's genuine dependency resolution.

**Q: Can humans override AI decisions?**
> Yes, that's the whole design. AI proposes actions, humans review and approve/reject in the "Pending Actions" panel. Nothing happens without human approval.

**Q: How accurate is the forecasting?**
> This is a demo with synthetic data. In production, accuracy depends on data quality and forecast period. The system uses simple linear regression + moving average (no ML libraries) so it's interpretable.

**Q: What happens if the backend fails?**
> The frontend shows friendly error messages. No crashes. If it times out, refresh the page.

**Q: Can we connect this to a real ERP?**
> Yes. The backend API is generic. You'd swap the data source from SQLite to your ERP's API.
